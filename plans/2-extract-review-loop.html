Extract review-loop into a standalone primitive skill

The annotate-over-tailnet review surface — serve an HTML artifact, let the human mark it up in-page, block until a verdict — is buried inside backlog. Any skill that needs a human to sign off on a rendered artifact wants it, so it should be an extracted primitive composed by name, not forked into every caller.

issue #2enhancementplan v2 — approved + path-prefix fix

Approach

The machinery is already ~90% artifact-agnostic — the extraction is mostly a move + generalize the shallow backlog-isms, plus a real skill wrapper (SKILL/README/references/evals/openai.yaml) modelled on the shipped staffing skill. The extraction is deliberately non-breaking: backlog keeps its own bundled copy and keeps presenting exactly as today; rewiring backlog to consume review-loop by name is a separate issue (see Out of scope).

File layout

New skill under skills/review-loop/, mirroring staffing's shape. Nothing is deleted from backlog.

The loop it preserves

Behavior is unchanged from today — only the home and the branding generalize.

User stories

  • As a skill author (maquette, prototype, a doc-review skill), I want to present any rendered HTML for human sign-off by invoking review-loop by name, so that I don't fork the review machinery into my skill.
  • As an agent, I want to serve an arbitrary artifact and block on a verdict-coded await, so that I proceed only on an explicit approve and revise on request-changes.
  • As Asher reviewing, I want to annotate the artifact in-page and get a batch verdict with hash-bound approval, so that I never approve a version I didn't see.
  • As Asher away from the machine, I want the tailnet port-proxy public URL and a repo-scoped hub listing everything awaiting me, with a local /hub fallback, so that reviews are findable from my phone.
  • As a maintainer, I want the old copyable pattern superseded by a pointer to this skill, so that future skills extract the primitive instead of copying it.
  • As a Codex user, I want the skill to present via agents/openai.yaml, so that it works when installed into Codex.

Implementation decisions

  • Standalone skill at skills/review-loop/, shaped like staffing: SKILL.md + README + reference/ + scripts/ + templates/ + agents/openai.yaml + evals/.
  • Scripts become the canonical home. review-server.py is generalized (per the before→after table); review-await.py, chrome.css/js move verbatim (already generic — chrome renders kind dynamically). Stdlib-only Python 3 preserved. No "Adapted from" header — as the extracted primitive, review-loop is the source of truth.
  • Contract split into two references. reference/review-loop.md carries the annotate→revise→approve loop (serve-time chrome, batch verdicts, await gate, ledger, hash-binding); reference/surface-and-hub.md carries one-surface-one-root, publish-don't-fork, local fallback, and the hub. Both are artifact-agnostic rewrites of backlog's presenting.md §§ Contract/Review loop/Hub. A third reference/scripts.md documents the CLI so callers can invoke the bundled scripts.
  • Plan skeleton is one example caller, not baked in. It ships under templates/ to demonstrate the stable-id contract the chrome anchors to; the server serves any HTML with stable ids and does not require it.
  • Three-part dependency surface declared in SKILL.md: bundled references (its own contract incl. the scripts) · project playbooks (a docs/agents/ surface-config playbook; on this repo it resolves to the existing environment.md § Presenting) · sibling skills (none — review-loop is a root primitive).
  • Supersede, don't delete-and-break. docs/patterns/review-surface.md is replaced by a short pointer to the review-loop skill per AGENTS.md § Conventions ("copy a pattern; extract a primitive"); the docs/patterns/README.md table row is updated. Existing forks (backlog, goodwork) are noted as migrating via their own rewire issues.
  • Backlog rewire is deferred. Backlog keeps its bundled scripts and presenting.md and works unchanged; consuming review-loop by name (and deleting backlog's copy) is a separate issue, so this extraction ships non-breaking.
  • Path-prefix submit fix (folded in during plan review). The chrome posts to root-absolute URLs (fetch('/event'), fetch('/version')), which resolve to the domain root and 404 when the page is served under a tailnet path prefix (/asher-skills/<n>/review) — the real deployment shape. The extracted chrome.js builds every client→server call mount-relative: derive a base from window.location.pathname (the directory of the served page) so requests resolve correctly whether served at / (loopback) or under a prefix; the /hub fallback link is made mount-relative too. Confirmed empirically that tailscale serve --set-path strips the prefix before forwarding (backend sees /event), so the client fix alone restores submit; additionally the server matches its endpoints (/event, /version, /hub, doc-root) suffix-tolerantly so a proxy that instead preserves the prefix is also handled. Backlog's bundled copy carries the identical bug and is deliberately not touched here (stays non-breaking per ac-12); it inherits the fix only when the deferred #6 rewire makes backlog consume review-loop.
  • Staffing: plan authorship is orchestrator work (this thread). Build-out delegates — the mechanical review-server.py generalization + file moves to the backend builder (gpt-5.5 via codex exec); the taste surface (SKILL/README/reference copy, skeleton header, superseded pattern doc) to a Claude builder (opus-4.8/fable-5). Verify/probe checking to the checker role.

Definition of done

Tags: runtime a genuine python3 exercise of the shipped scripts · file a file/content check · probe a dry-run eval graded against an answer key.

  • The skill exists at skills/review-loop/ with SKILL.md, README.md, at least the three reference/ docs, scripts/review-server.py, scripts/review-await.py, scripts/pages/chrome.{css,js}, a templates/ caller skeleton, agents/openai.yaml, and evals/probes.md.file
  • SKILL.md declares all three dependency-surface kinds explicitly — bundled references (incl. the scripts), project playbooks (a docs/agents/ surface-config playbook), and sibling skills (states "none — root primitive").file
  • Artifact-agnostic kind: review-server.py --kind maquette --doc <any.html> --title T --surface <tmp> --port 0 starts and prints JSON with url/hub/port; a non-plan/prototype kind is not rejected.runtime
  • Serves arbitrary HTML, committed file stays pure: for a minimal <html><body><p id="x">hi</p></body></html>, GET / (with token) returns the body with __REVIEW_BOOTSTRAP__ + chrome css/js injected before </body>, and the on-disk file is byte-identical afterward.runtime
  • Verdict-coded await: posting approve (matching hash) → review-await.py exits 0; approve_with_nits → 3; request_changes → 10; no event before deadline → 124.runtime
  • Hash-bound approval (load-bearing invariant): an approve POST whose doc_hash ≠ the current document hash returns HTTP 409 {"error":"stale"}; editing the doc after a page load invalidates a prior-hash approval.runtime
  • Hub + local fallback: on start registry.json and index.html are written under the surface dir and the entry is removed on clean exit; GET /hub returns the generated index; --sweep drops dead entries and prints {"swept":[…]}.runtime
  • No baked-in backlog/plan identity: the server docstring and the hub H1 carry no hardcoded "backlog"; the chrome copy reads neutrally for any artifact kind; the plan skeleton is shipped as a template, not required by the server (proven by ac-4 serving a skeleton-free doc).file
  • Pattern superseded: docs/patterns/review-surface.md is replaced by a short pointer to the review-loop skill (compose by name), and docs/patterns/README.md's table row is updated per AGENTS.md § Conventions.file
  • agents/openai.yaml is present and parses as valid YAML, presenting the skill for Codex per docs/patterns/codex-compat.md.file
  • Probe eval passes: evals/probes.md holds situated dry-run probes with an answer key; both a Claude executor and a gpt-5.5 (codex exec) executor route correctly on the core probes — e.g. "sign off on a rendered maquette" → serve via review-server + block on review-await (don't reinvent); "no surface recorded" → local fallback; "changes requested" → revise + write ledger before re-await. All core probes pass on both executors.probe
  • Non-breaking: backlog's own scripts and presenting.md are untouched and it still presents as before; the rewire to consume review-loop by name is explicitly deferred and noted in the superseded pattern doc.file
  • Submit works under a path-prefix mount (the real bug): (a) runtime — with the review server behind a simulated stripping proxy that mounts the doc at a prefix (e.g. /asher-skills/2/review/), POSTing a verdict to the prefixed /event path is accepted and the verdict-coded review-await.py exit still fires (0/3/10); (b) file — chrome.js contains no root-absolute fetch('/event' or fetch('/version', and constructs those URLs from a mount base derived from location. Both loopback-root and prefixed serving submit successfully.runtimefile

Test seams

  • Script CLI (highest-value seam). ac-3 through ac-7 drive review-server.py and review-await.py directly with python3 over loopback — real HTTP GET/POST, real event log, real registry files in a tmp surface dir. This is the load-bearing behavior and it is genuinely runtime-checkable.
  • Path-prefix mount (ac-13) — the seam the v1 checks missed. Loopback-root POSTs never exercise the prefix that broke submit in production. The v2 runtime test puts a tiny stdlib stripping proxy in front of the server, mounts the doc under a prefix, and POSTs the verdict to the prefixed path — asserting the verdict-coded await still fires. Paired with a static assertion that chrome.js builds mount-relative URLs.
  • File/content checks. Layout, dependency-surface declaration, neutral branding, superseded pattern, openai.yaml — grepped/parsed against the tree.
  • Probe eval. The skill's routing value (does an author invoke it correctly, honor fallback, write the ledger) is judged by dual-executor dry-runs per docs/patterns/probe-evals.md.
  • Deliberately untested: the in-browser chrome interaction (drag-to-annotate, auto-reload) — no headless browser in this repo; it moves verbatim and is covered by the human's own review pass on the tailnet surface, not automation.

Evidence required

  • Terminal transcript of the script-behavior checks (ac-3 – ac-7): server start JSON, an injected-chrome GET, the three verdict exit codes + timeout, the 409 stale path, and a --sweep run.
  • The dual-executor probe transcript + a pass/fail table mapping each probe to its answer-key criterion (ac-11).
  • A screenshot of a non-plan artifact (e.g. a trivial doc) served with chrome, rendered on the surface — proof of artifact-agnostic serving.
  • Terminal transcript of the path-prefix submit test (ac-13): a verdict POSTed through a prefixed mount, the accepted response, and the matching verdict-coded await exit — proof the production failure mode is closed.
Risks
  • Transitional duplication: two copies of the scripts (backlog + review-loop) until the rewire issue lands. Mitigation: the superseded pattern doc names backlog/goodwork as pending-migration forks so the duplication is intentional and tracked, not drift.
  • Over-generalizing the server introduces a regression in the exact loop backlog depends on. Mitigation: the changes are surgical (kind/branding/scope only); the runtime criteria exercise the invariants (hash-binding, verdict codes, sweep) that a regression would break.
Out of scope
  • Rewiring backlog to consume review-loop by name and deleting backlog's bundled copy — a separate rewire issue (kin to the #6 composer rewire).
  • Migrating goodwork's multi-page workspace variant onto review-loop — its own issue; review-loop's first cut is the single-document loop.
  • A setup command that installs the surface-config playbook into a fresh repo — for this repo the playbook already exists as environment.md § Presenting; a standalone installer is future work.
  • Any change to the tailnet/surface config in docs/agents/environment.md.
