Implemented build contract · revision 9 Issue #61 Machine reconcile deferred until after merge

Harness-specific skills and deferred global context

Keep one public contract, compile only the guidance each harness needs, and move cold machine policy behind sharp, fail-safe pointers.

Decision summary

Two linked, separately shippable tracks

#61 owns installed skill variants. A companion issue owns global pointer modules and token budgets. They share a design principle, not a migration transaction.

Symlink remains the default

Unvaried skills keep one real .agents package plus the Claude symlink. Only a declared variant earns two real harness trees.

Hot kernel, cold modules

Global files keep exact triggers and safe failure behavior. Harness-specific policy lives in owner-written modules loaded only when a trigger fires.

Harness capabilities route before models

browser-use, computer-use, and imagegen belong to installed harness/tool providers. Choose a working provider route first; rank only models eligible to operate it.

Reachability is directional, not native-only

Codex can hand bounded worker tasks to effect-verified Claude Code routes. Its effective roster therefore includes verified Claude models, with probed CLI aliases and native-wrapper mechanics.

External work stays visible in the native tree

Every cross-harness CLI dispatch travels through a named, watched native relay staffed by the cheapest native model allowed by the floor. The parent owns judgment, the external harness owns the task, and the wrapper owns lifecycle and verbatim relay only.

Design consequence

Both directions prove native-tree observability, bounded lifecycle, and raw relay. Claude→Codex also proves wrapper-model selection and self-report: codex exec takes -m for the external model and the native Agent tool takes a model parameter for the wrapper. Codex→Claude retains one red acceptance seam because the tested native spawn interface did not expose wrapper-model selection or assignment. Adopt the symmetric wrapper contract with its fail-safe: where native spawn cannot select or report the wrapper model, preserve observability, record the staffing gap, and do not claim floor/cost compliance.

Image generation and taste remain separate

Stories

  1. As a leaf worker, I want cold machine policy omitted, so that my context is spent on the assigned work.
  2. As an orchestrator, I want a sharp pointer to the complete active-harness roster, so that routing stays correct without eager context cost.
  3. As a skill author, I want one public dependency and setup contract with small harness overlays, so that variants cannot drift into unrelated implementations.
  4. As an auditor, I want declared provider divergence distinguished from accidental copies, so that real drift remains actionable.
  5. As an image creator, I want image generation routed to a Codex provider and reviewed separately for taste, so that capability and quality gates do not deadlock each other.
  6. As a task router, I want harness capabilities supplied by named skills/plugins with recorded fallbacks, so that model ranking never invents tool access.
  7. As an orchestrator, I want every external-harness worker represented by a cheap named native relay, so that the native agent tree shows who is working, on what, and through which route.

Definition of done

  • Canonical ownership is enforceable: setup-asher-skills alone owns convention pointers/modules; staffing alone owns roster pointers/modules; either owner’s reconcile leaves the other owner’s bytes unchanged.
  • Each harness global file contains exact on-demand triggers for routing and presentation, uses no eager import, names an absolute module path, and defines safe behavior when that module is unreadable.
  • All Asher-owned always-loaded global text measures at or below 140 o200k-equivalent tokens per harness; each pointer stanza is at or below 65 tokens, with raw-character gates in stdlib-only tests.
  • A direct leaf task that makes no routing, capability, watcher, fallback, or presentation decision does not read either deferred module.
  • Routing, child/worktree creation, browser/computer/Imagegen work, watcher assignment, route loss, and presentation each read the correct active-harness module before acting; project deltas apply after the global staffing module.
  • Each deferred module is self-contained and contains no runtime branch for the other harness; a missing staffing module blocks dispatch, while a missing presentation module preserves the local artifact and blocks publishing.
  • The compatibility contract records browser-use, computer-use, and imagegen against named harness/tool providers—not model rows—and ranks models only after selecting a reachable provider route.
  • A user-facing image workflow separates Codex-provided generation from human or taste-qualified review; route loss reports an image-generation capability gap rather than substituting browser/computer use.
  • A skill without variants retains the current real .agents primary and expected Claude symlink, with unchanged lock and audit behavior.
  • A declared variant compiles one shared public identity, invocation/execution policy, dependency closure, external contract, and setup owner into correct self-contained real trees for each active harness; overlays that change shared fields are rejected.
  • Variant provenance records the shared source revision plus provider identity and effective-tree hash; install effect-verifies mount type, provider, content, and lock state rather than trusting command exit status.
  • Audit passes valid declared divergence and emits distinct findings for missing provider mount, wrong provider, altered tree/hash, undeclared independent copy, and shared-contract drift.
  • The pilot skill contains zero branches intended only for the other harness and reduces runtime-loaded text by at least 20% in each harness, measured against the current unified package.
  • Migration is module-first, atomic, harness-aware, reversible by proposal rather than deletion, and idempotent: it recognizes and replaces only setup’s seeded legacy header/## Conventions block, refuses an unowned block, preserves unrelated bytes, and a second setup/reconcile makes zero filesystem or lock changes.
  • The provider registry records Codex ChatGPT-in-Chrome as the primary browser route, project-recorded agent-browser as its fallback, Codex Computer Use with no substitute, and native Codex Imagegen with repo codex-imagegen as the headless fallback.
  • Claude Code records no native provider for these three capabilities; any Claude-initiated browser or image work names the machine fallback or dispatched Codex provider explicitly, and never attributes the effect to Claude.
  • The applied contents of the six effective machine files are byte-identical to the complete payloads in this plan, with the two presentation modules intentionally identical.
  • Application begins a fresh transaction barrier, writes and verifies all four deferred modules, preflights both global files before reconciling either, applies both Presentation sections before either Staffing section, verifies all four final sections, and removes the barrier. A stale or partial run cannot satisfy the gate; preflight failure preserves both working inline policies.
  • The Codex-effective roster includes effect-verified Claude worker rows, keeps bounded CLI workers out of coordinator eligibility, maps roster display names to probed CLI aliases, and requires a watched native wrapper for every Codex→Claude invocation.
  • Every cross-harness dispatch runs inside a watched native wrapper whose name identifies the external harness/model and task, whose native staffing is the cheapest model allowed by the current floor, and whose role is limited to process supervision and raw relay. The native spawn seam must accept the resolved wrapper model or report the assigned model; otherwise floor/cost compliance remains unproven.

Open Codex seam

Native spawn can enforce or expose wrapper staffing

Claude→Codex proved wrapper-model selection and self-report. Codex→Claude proved naming, watching, bounded execution, and raw relay, but its tested spawn interface exposed neither model selection nor assignment. Keep floor/cost compliance red in that direction until the seam exists.

Risks and mitigations
  • Pointer skipped: shadow phase, fail-closed text, real-harness probes, and retained inline policy until pass.
  • Third memory surface: generate modules inside each harness directory from one source; do not add a new auto-loaded memory system.
  • Two implementations drift: overlays cannot declare shared identity/dependencies/setup; compiler rejects changes.
  • Tree/lock split-brain: stage and publish trees plus lock records atomically; injected-failure tests.
  • Variants everywhere: require measured provider branching and ≥20% pilot saving; otherwise retain the symlink.
  • Capability attached to model: select a named harness/tool provider before executor or model ranking, then effect-verify that route.
  • Adapter-skill sprawl: reuse the installed Codex plugin/system skills and repo codex-imagegen; create an agent-browser adapter only when a probe proves durable procedure is missing.
  • Stale Codex roster: issue #59’s three-state directional audit and separately consented global-base correction prevent a working Claude route from remaining hidden.
  • Unverifiable wrapper cost: require native spawn to select or report the wrapper model. Claude→Codex passed via the Agent model parameter and subagent self-report; Codex→Claude remains unproven at this seam. Preserve agent-tree observability, record the gap, and never claim the floor/cost rule passed where selection or assignment is hidden.
