Backlog orchestration: liveness, staffing & browser-safety fixes

Backlog runs go dark — work parks silently, status takes nudges, workers are hired without the permissions or the model their role needs, and browser fallbacks reach into the user's own machine. Observed on Codex; the same seams exist when runs go through Claude Code. This plan closes them harness-symmetrically.

The shape — why runs went dark

One root cause carried three symptoms: supervision dies at turn boundaries — completions land durably, but nothing owns waking on them, so the human becomes the wake mechanism.

Run state vocabulary becomes exactly five states, used in every user-visible report:

Why this holds for Claude Code too

Every failure class maps to a seam that exists in both harnesses; each fix below is written against the role, and each staffing variant binds it to its harness's mechanics. No-burden rule: a workaround earns its cost only where its platform gap exists. The contracts state the invariant ("a verified wake path exists before the turn ends"); each harness binding names its native wake sources — on Claude Code, tracked background tasks and subagent completion notifications already re-invoke the session, so a Claude-led run satisfies the contract natively and inherits none of the bounded-polling machinery Codex-led runs need. Only genuinely untracked waits (fire-and-forget shells, external CI, review verdicts) carry the explicit-ownership cost on either harness.

Workstreams

W1 — Liveness & reporting contract (backlog: run.md, issue-loop.md, run-state.md)

The five-state vocabulary, heartbeat, wake-path precondition, and first-class waits become contract text (rendered in Updated copy). run-state.py gains a completion gate: a stale projection or missing handoff.md refuses terminal “Done”.

W2 — Queue invariant & runnable waves (backlog: run.md)

Persist the authorized set at dispatch; claim exactly what can be staffed and supervised right now. No artificial numeric cap on subagents — the bound is real capacity: available harness slots, session budget, and a live supervisor per claim. An issue with no runnable worker stays ready-for-agent instead of sitting claimed (the iv2 run held 7 claimed issues against 4 slots for ~18 h). Terminal reports enumerate every authorized issue or are invalid.

W3 — Cross-harness dispatch: permission envelope, effect probes, staffing telemetry (staffing skill)

One policy, both directions: the hiring orchestrator owns the envelope. Whoever spawns a worker is responsible for granting — in the dispatch command itself — every permission the job needs, proving it with a reversible probe, on the model the roster staffed, recorded where resume can audit it. For now the envelope is yolo both ways (Asher, review round 4): each harness's full-permission mode, matching how the orchestrators themselves run; the scoped role envelopes below stay documented as the hardening path.

Also in W3: wrapper purity (a wrapper supervises and relays only — never repurposed to edit), minimal-context child spawns (no full-history forks), and the roster assertion that would have kept pipelines wave 1 off terra@medium.

W4 — Browser policy inversion & Computer Use gate (staffing + backlog setup + project playbooks)

Surfaces: both staffing variant modules, machine-audit.md + rankings-and-routing.md examples, backlog setup.md step 6 + templates/common/environment.md, regenerated deployed machine modules, and a reconcile pass over metis/pipelines/integrations-v2 playbooks (superseding the uncommitted iv2 edits).

W5 — Delivery/merge boundary (existing issue #70)

Scrub “to a merged PR” from all high-level backlog copy; backlog ends at review-ready PRs; a new small merge-changes skill is the explicit approval gate (re-verifies checks at merge time, merges in dependency order, stops on semantic conflicts). Specified in #70 — this plan schedules it.

W6 — Review-loop verdict durability & custody

One canonical durable event path; verdict events consumed on resume without a live watcher (the two orphaned overnight approvals); events copied into the run root before cleanup; token custody split (orchestrator mints, thread awaits) plus client evidence per event, per retro #46's self-approval finding.

W7 — Regression probes (dual-executor, per docs/agents/probe-evals.md)

Every behavior this plan changes gets a situated probe run against both executors (Claude subagent + codex exec) with a prewritten answer key — see acceptance criteria.

W8 — Codex platform reports (deferred; not filed upstream)

Deferred per Asher (post-approval, 2026-07-17). The reports are written up in issue #77 — child completion should wake a finalized parent; list_agents must not omit live children; the collaboration slot cap should be introspectable; unexplained 47–65 m event gaps need telemetry; resume --last is racy under concurrency — and are not filed to the Codex tracker until Asher says so. The contracts above are written to survive these staying unfixed, so nothing downstream waits on this.

Staffing, Codex variant — Mechanics (the opus fix)

Current (one line, no permission envelope)

“It runs claude -p --model <verified-alias> '<self-contained prompt>' </dev/null, never adds --bare, and captures the durable result.”

Proposed

The envelope policy is bidirectional and orchestrator-owned: whoever hires a worker grants, in the dispatch command itself, every permission the job needs. Per-harness syntax, one responsibility. Prompt text never grants permissions; only flags and settings do. Current machine policy (Asher, plan review round 4): yolo both ways for now — matching how the orchestrators themselves run:

# Codex → Claude — worker, run from the target worktree (yolo, for now):
claude -p --model <verified-alias> --dangerously-skip-permissions \
  '<self-contained prompt>' </dev/null

# Claude → Codex — worker (yolo, for now):
codex exec --cd <worktree> --sandbox danger-full-access \
  '<self-contained prompt>' </dev/null        # + explicit generous timeout

# Hardening path (documented, not active): role-scoped envelopes —
#   Codex→Claude:  --permission-mode acceptEdits --allowedTools 'Edit' 'Write' 'Bash(git *)' …
#   Claude→Codex:  --sandbox workspace-write -c sandbox_workspace_write.network_access=true
#   reviewers/checkers: read-only sandbox / no mutation flags.
  • Effect-class probe first. Before the first substantive dispatch on a route, run a reversible probe matching the role: a one-line file write for a builder, a read for a reviewer. A denied effect quarantines the route directionally (build dead ≠ review dead) and reroutes immediately. A text-only echo probe verifies nothing about effects.
  • Session identity. Capture the worker's session id at launch; resume by id, never resume --last — parallel wrappers collide on it.
  • Wrapper purity. A wrapper supervises the bounded process and relays raw output and lifecycle status. It is never repurposed to edit; if capacity blocks a real worker, free, queue, or report blocked.
  • Telemetry. Record the spawned model, effort, role, route, and session id in the run-state spawn event and assert model+effort against the staffed role before dispatch; a mismatch is a dispatch blocker.
  • Minimal context. Children get a self-contained task packet (issue, role, worktree, gate, expected return) — never a full conversation-history fork.

Backlog run.md — new contract blocks

Proposed (added to steps 5–6)

Liveness contract

  • Report state in exactly one of working | waiting | blocked | stalled | complete. Announce every phase transition to the user.
  • While any issue is active, emit a heartbeat at most every 10 minutes: phase, last durable effect, next expected event.
  • Never end a turn while children, gates, or CI you own are live unless a verified wake path exists (harness notification, tracked background task, scheduled wakeup). Where none exists, hold a bounded foreground wait loop. “Dispatched” / “resumed” may be reported only after the wake path is verified.
  • Every wait is recorded with owner, object, expected event, next-check deadline, and wake path. A prose-only wait (“awaiting CI”) is a contract violation. Terminal CI waits belong to the orchestrator, not a parked issue thread.

Queue invariant

  • Persist the authorized issue set at dispatch. Claim exactly what can be staffed and supervised now — bounded by real capacity (harness slots, session budget, a live supervisor), never an arbitrary numeric cap; the rest stay ready-for-agent — claimed-but-unworkable is forbidden.
  • The terminal report enumerates every authorized issue as completed | blocked(edge) | deferred(wave) | returned | interrupted. “Done” with an unaccounted issue is invalid. Write handoff.md and pass the run-state verify gate before finishing; a stale projection refuses completion.

Backlog verify.md / issue-loop.md — grading language

Proposed

Every per-criterion verdict carries a grade: live (exercised on the real surface) | static-substitute (inspection or approved substitute — named) | not-run | blocked. A checker that cannot execute records static-substitute or blocked — never unqualified PASS — and coordinator/orchestrator reports quote the grade verbatim. Acceptance requires every criterion live or carrying its pre-approved substitute as a caveat.

How it will be checked

Three seams, lowest count that proves the criteria: source reads (ac-1, 2, 11, 12 — the policy text is the artifact), dual-executor situated probes with prewritten keys per docs/agents/probe-evals.md (ac-3, 5–9, 12, 13), and live smokes where only reality can answer (ac-4 permission envelope, ac-10 run-state gate via its test file, ac-14 setup audit, ac-15 next-run transcript). Deliberately unchecked: the Codex platform asks (W8) — we can't test their fixes, only that our contracts survive without them.

Risks & mitigations

  • Heartbeat spam — a 10-minute cadence could get noisy on long runs. Mitigation: heartbeat only while issues are active, state-change-first wording, and the cadence is a playbook-tunable default.
  • Yolo blast radius — full-access workers in both directions widen what a bad worker can touch. Accepted for now (Asher, round 4): workers run inside isolated worktrees, the effect probe bounds route surprises, adversarial review gates every diff before merge, and the scoped role envelopes stay documented as the hardening path to adopt if this bites.
  • Contract text that agents ignore — the Jul 15–16 runs ignored run-state.md's existing rules. Mitigation: the load-bearing rules become mechanical gates (run-state verify refuses completion; probes regress the rest) rather than prose alone.
  • Cross-repo drift — consumer playbooks (metis pins Chrome today) lag the source change. Mitigation: ac-14's reconcile pass is in the definition of done, not a follow-up.

  • Merging any PR — humans (or an explicitly invoked merge-changes) merge; this plan never does.

  • Changes to the model roster's rankings themselves — only telemetry, assertion, and provider routing change; who ranks where does not.
