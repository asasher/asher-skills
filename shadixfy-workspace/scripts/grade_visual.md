# Blind visual judge — prompt template

Use this to score outputs holistically across conditions, beyond the mechanical assertions.
Per the methodology, **judge blind**: don't tell the judge which screenshot came from which
condition. The harness folder names reveal the condition, so copy the screenshots to neutral
names first (e.g. `A.png`, `B.png`, `C.png`) and keep a private mapping.

## Setup

For one (eval, agent), gather the three condition screenshots:

```
iteration-1/dashboard/claude/no_skill/outputs/screenshot.png
iteration-1/dashboard/claude/uncodixfy/outputs/screenshot.png
iteration-1/dashboard/claude/shadixfy/outputs/screenshot.png
```

Shuffle them into `A/B/C` and note the mapping somewhere the judge can't see.

## Judge prompt

> You are a senior product designer reviewing three versions of the same screen
> (an analytics dashboard / a settings page / a pricing page — state which). They were
> generated by AI. Score each image independently, 1–5, on these axes, then pick an overall winner:
>
> 1. **Restraint** — does it avoid the "an AI made this" look (gradient/glass decoration, glow,
>    oversized rounded corners, hero blocks where they don't belong, eyebrow kickers)?
> 2. **shadcn-ness** — neutral palette, hairline borders separating surfaces, modest radius and
>    shadow, sober typographic hierarchy. Does it read like a shadcn/ui product surface?
> 3. **Hierarchy & legibility** — is structure clear without decoration doing the work?
> 4. **Would-ship** — would a real team ship this internal/marketing surface as-is?
>
> For each image give the four scores with one sentence of evidence each (reference what you see).
> Then name the overall winner and explain in two sentences. Do not assume any image "should" win.

## Recording

Save scores to `iteration-N/visual_scores.json`, e.g.:

```json
{
  "dashboard/claude": {
    "no_skill":  { "restraint": 2, "shadcn": 2, "hierarchy": 3, "would_ship": 2 },
    "uncodixfy": { "restraint": 4, "shadcn": 3, "hierarchy": 4, "would_ship": 3 },
    "shadixfy":  { "restraint": 4, "shadcn": 5, "hierarchy": 4, "would_ship": 4 },
    "winner": "shadixfy",
    "notes": "shadixfy kept the token-driven neutral surfaces; uncodixfy was clean but warmer/less systematic; no_skill had a gradient KPI grid."
  }
}
```

## Running the judge with Claude (optional, automatable)

Claude can view images directly. From the workspace you can do, for one eval/agent:

```bash
claude -p "$(cat scripts/grade_visual.md | sed -n '/## Judge prompt/,/## Recording/p')" \
  --permission-mode bypassPermissions \
  iteration-1/dashboard/claude/no_skill/outputs/screenshot.png \
  iteration-1/dashboard/claude/uncodixfy/outputs/screenshot.png \
  iteration-1/dashboard/claude/shadixfy/outputs/screenshot.png
```

(For a true blind run, copy the three to shuffled neutral filenames first and pass those.)
