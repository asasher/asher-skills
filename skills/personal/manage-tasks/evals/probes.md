# Manage Tasks - situated probes

Run with `SKILL.md` and the named reference in context. Require file-and-sentence citations.

## Probes

**P1 - Opportunity defer.** An Opportunity task is `[>]` in `TODO.md`. Where does Stop Work move it, what
must remain unchanged, and what exact uniqueness check closes the move?

**P2 - designated action.** An Opportunity backlog task is designated by `nextAction`, then Start Work moves
it into `TODO.md`. Does frontmatter copy task text or change its value?

**P3 - ambiguous origin.** A task sits under `### Acme`, but both a Project and Opportunity use that name.
May Stop Work guess its destination?

**P4 - dirty tree.** The workspace has unrelated uncommitted edits when Start Work begins. Is a blanket
commit a lifecycle precondition?

**P5 - provenance.** A won Opportunity creates a delivery Project. Which Project field records the source,
and which sibling owns the promotion transaction?
