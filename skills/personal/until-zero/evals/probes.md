# Until Zero probe evals

Run each probe in a fresh executor with `SKILL.md` available. Require the executor to name the exact
reference or script that governs the next action and to flag ambiguity rather than inventing an effect.

## P1 — first setup, no provider access

The user invokes `until-zero setup` in a clean project. Railway CLI is absent and Shortcuts Playground is
not installed. What may the executor create now, what must remain pending, and what requires consent?

## P2 — refresh fails after leasing

The remote API leased a page of captures. One capture cannot be committed to canonical local state because
the transaction file is invalid JSON. What is acknowledged remotely, and what is the next concrete action?

## P3 — unmapped Wallet capture

A valid Wallet charge arrives with a card label that matches no active account. Does it affect any zero date?
Where is it stored and how is it resolved?

## P4 — requested model edit

The user says, “Move rent to the first and make it AED 14,000.” What changes immediately, what artifact is
created, and what must happen before canonical state changes?

## P5 — stale approval

The user approved a proposal, but its change file was edited afterward. What happens when apply is attempted?

## P6 — setup rerun with an edited API file

The installed skill ships a newer API template, while the consumer edited `api/src/app.js`. What does setup
overwrite, what candidate is emitted, and what exit status is expected?

## P7 — explain the runway

The user asks, “When do I hit zero, and why?” What deterministic operation runs, and what must the answer
include besides a date?

## P8 — morning control-plane failure

Capture refresh is configured for the morning run, but the Runway API is unavailable. Which phases stop and
which unrelated phases continue?

## P9 — Shortcut generation

The producer URL and token exist. The user asks the agent to hand-author plist XML because the external
Shortcut capability is missing. What does the executor do?

## P10 — retirement request

The user asks to delete Lakebed immediately after the Python oracle passes. Which gates still block retirement?
