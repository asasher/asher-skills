# shape

The shaping stage of the SDLC family: turns an idea, an issue, or a batch of issues into
execution-ready work. Composes `interview` (the questioning method), `domain-modeling` (terms and
ADRs), `to-spec` (synthesis), and `to-tickets` (projection), with `research`, `prototype`,
`staffing`, and `review-loop` as optional probes and gates.

A stateful composite: everything settled lands in durable artifacts (issue threads, glossary
entries, ADRs, specs, tickets), and a resumed session picks up from the record, not from memory.
