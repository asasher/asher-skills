# odysseys — three alternative five-year work futures

Adapted from Designing Your Life's Odyssey Plans. Purpose: generate real options, not pick the one true calling. Requires a profile; read `goodwork/PROFILE.md` first.

## Build three plans

Draft three genuinely different five-year paths **from the profile's evidence**, then refine each with the user:

1. **Plan 1 — the current thing, done well.** The evolved version of the path they're on (or the obvious next role). Not a strawman: make it the best honest version.
2. **Plan 2 — the adjacent thing.** What they'd do if Plan 1 vanished tomorrow. Usually built from the anti-spark inversion and the strongest under-used strength.
3. **Plan 3 — the unconstrained thing.** What they'd do if money, image, and others' judgment didn't matter. This one surfaces suppressed data; it is not automatically the "real" answer.

For each plan:
- A six-word title.
- A year 0–5 sketch: the work, the people, where the money comes from, one milestone per year.
- The two or three questions this plan would answer about their life.
- **Gauges (1–5): Resources** (time, money, skills, network to pull it off), **Excitement** (honest affective pull), **Confidence** (belief they could test it), **Coherence** (fit with the profile's snapshot, values, and whole-life dashboard).

## Rules

- Plans must diverge in kind, not salary band. If two plans share the same daily activities, regenerate.
- Use the profile: every plan should name which energizers it feeds, which anchor it honors, and which constraint it respects. A plan that violates a gravity constraint is fantasy, not a plan.
- Include at least one plan compatible with the decent-work floor and risk budget as they exist *today*.
- Portfolio shapes count: a plan can be three small bets, not one job title.
- Watch the excellence trap: if all three plans extend what they're rewarded for but scored low on excitement, force one plan from the "almost too easy to count" strengths.

## Choosing well (when they ask "so which one?")

Don't rank-order into a verdict. Instead:
1. **Gather** — the plans, the gauges, the profile.
2. **Narrow** — usually the real choice is between two, or a hybrid.
3. **Choose** — best **doable** option beats best theoretical option. Check gut reaction as data, coherence as tiebreaker.
4. **Let go** — once a direction is chosen and tested, stop re-litigating. Decision quality is not outcome quality.

But before any big commitment: the next step is **never** "quit and leap" — it's `prototype`. Each plan yields two or three cheap tests; run those first.

## Output

Write `goodwork/ODYSSEYS.md`: the three plans with gauges, the open questions each would answer, and the top prototype candidates per plan. End by handing off to `prototype`.
