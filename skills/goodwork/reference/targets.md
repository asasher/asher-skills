# targets — build and rank the target list

Steve Dalton's LAMP method, adapted. The unit of a good search is the employer/target, not the posting. Requires the profile and, ideally, `NICHE.md`.

## Build the list (speed over perfection)

Generate **40+ targets in one sitting** — employers, but also, when the profile points that way, fellowships, grant-funded projects, agencies/studios, open-source orgs with paid roles, or client segments for independents. Sources: the niche's community map, employers of people the user admires, competitors/customers/vendors of past employers, "best places to work in [niche]" lists, conference sponsor pages, portfolio pages of relevant VCs, who's-hiring threads. Verify each against current sources — check the org still exists, still does the thing, and note anything material (layoffs, pivots, hiring freezes).

List first, judge later — premature filtering kills the list's value.

## Score it (LAMP)

| Column | Meaning | Score |
|---|---|---|
| **L**ist | The target's name, plus segment/size/location | — |
| **A**dvocacy | Do we have or can we plausibly get an insider? (alum, past colleague, community member, 2nd-degree) | 1–3 |
| **M**otivation | Honest gut pull toward working there — score fast, first instinct | 1–5 |
| **P**osting | Live relevant opening (3), adjacent opening or regular hiring (2), nothing visible (1) — nothing visible is not disqualifying; it means outreach-first | 1–3 |

Sort by Motivation, then Advocacy, then Posting. Motivation leads because the profile's whole point is knowing what's worth wanting; a reachable, posting-rich target the user dreads is a bad target.

Then run the **profile veto pass** over the top 15: strike any target that violates a dealbreaker, the decent-work floor, or a gravity constraint, and say why — this is where the Define phase pays off.

## Work the Top 10 as a queue

- Actively work the **top 5** first: research each enough to name why-them-specifically, find two or three insider candidates, and open outreach (hand off to `outreach`).
- Targets 6–10 warm up next. Below 10 is bench.
- **Replace, don't hoard**: when a target exhausts (no response after full cadence, bad listening-tour signal, freeze), retire it with a reason code and promote from the bench. Keep the working set at 10.
- Refill trigger: bench below 10 → schedule a 30-minute list-building block via `daily`.

## Output

Write `goodwork/TARGETS.md`: the full scored table, the Top 10 with why-them notes and insider candidates, retired targets with reason codes. Register the Top 5 in `PIPELINE.md` and hand off to `outreach`.
