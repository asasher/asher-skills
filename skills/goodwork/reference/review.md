# review - the weekly loop

Thirty minutes, once a week, same day each week. The review closes the loop between the campaign (what happened) and the profile (what it means). It reads `pipeline.json`, `targets.json`, `leads.json`, `metrics.json`, `evidence-inbox.json`, `JOURNAL.md`, `EXPERIMENTS.md`, `NICHE.md`, and `PROFILE.md`.

## 1. The numbers (5 min)

Pull the week's metrics from the pipeline: outreach sent, response rate, conversations, referrals, applications, screens, interviews, offers, source-of-interview. Compare to quota and to last week. Then run the funnel diagnosis from [pipeline.md](pipeline.md) — find the narrowest constriction and name which command fixes it. One diagnosis per week; don't change three variables at once or the next week's numbers mean nothing.

## 2. The energy read (5 min)

Scan the journal week: which days had real wins, what consistently energized, what consistently drained, where flow showed up. Two questions:
- Does the pattern match the profile's energy map? Contradictions are findings — route to `profile`.
- Is the *search design* fitting the person? (Drowning in cold outreach they hate while their community route sits unused is a design bug, not a discipline problem.)

## 3. The learning read (10 min)

- Listening-tour and conversation debriefs: is the market confirming or correcting the niche hypotheses? Every ~5 conversations, `NICHE.md` should visibly update. If it hasn't changed in three weeks of active conversations, either it converged (good — say so) or the debriefs aren't being mined.
- Experiments: which prototypes hit their time box this week? Run debriefs (per [prototype.md](prototype.md)), move profile claims to *tested*, kill or escalate each experiment explicitly.
- Reason codes on the week's rejections/retirements: three of the same code is a message.
- Evidence inbox: drain `evidence-inbox.json` into `PROFILE.md` using [profile.md](profile.md). Do not leave profile-relevant campaign evidence stranded in JSON.

## 4. Recommit or redesign (10 min)

- Set next week's quotas — tuned to the diagnosis, the user's real capacity, and honest urgency. Shrinking deliberately is fine; drifting is not.
- Refill triggers: bench targets, artifact queue, conversation list.
- **Monthly, additionally**: re-score the active odyssey plan's gauges (still excited? still coherent?), check the whole-life dashboard (what has the search itself been costing?), and ask the 80,000-Hours-style question — "If I were starting this search fresh today, knowing what I now know, would I design it the same way?" If no: change it, and log why in the profile changelog.
- **On offer or search-end**: run a full debrief into the profile — what the search proved about this person's good work — so the next season starts from evidence, not memory. The profile outlives the search; suggest a light `daily` journal habit continue into the new role for the first 90 days, feeding a six-month `profile` review.

## Output

A short written review appended to `goodwork/JOURNAL.md` (numbers, diagnosis, energy read, evidence inbox decisions, learnings, next week's plan), profile/niche updates applied, `metrics.json` and `evidence-inbox.json` updated, and the single most important change for next week stated in one sentence.
