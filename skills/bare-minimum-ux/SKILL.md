# Bare Minimum UX

Bare Minimum UX captures Asher's required UX standards for applications and web design.

## Current Status

This skill is intentionally a stub. For now, treat it as a signal to prioritize practical, polished, user-respecting UX decisions and to ask for or add concrete rules as they become available.

## Working Guidance

- Prefer usable product surfaces over marketing shells.
- Keep interactions clear, efficient, and hard to misuse.
- Watch for the basics: readable text, responsive layout, accessible controls, sensible defaults, clear feedback, and no broken or unfinished states.
- When adding future guidance, keep each rule concrete enough to apply during implementation or review.
