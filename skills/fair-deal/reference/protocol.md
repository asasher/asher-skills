# Protocol — file layout, state machine, turns, sync

This is the shared source of truth. Every subcommand obeys it. The whole system is two clones of one private
GitHub repo, coordinating only through committed files. There is no direct channel between the two agents.

## Repo layout (committed unless marked)

```
<deal-repo>/
  README.md                  # human-facing: what this is, how to take part
  canvas.html                # the review surface — open in a browser, Import canvas.json to view
  canvas.json                # SHARED source of truth: the filled Fair Deal Canvas
  AGREEMENT.md               # generated by `draft` when Ready (absent until then)
  negotiation/
    state.json               # the state machine (see below) — the coordination spine
    log.md                   # append-only human-readable negotiation log (one block per move)
    from-A/                  # Party A agent's outbox: round-N.md argument files (only A writes here)
    from-B/                  # Party B agent's outbox: round-N.md argument files (only B writes here)
  .claude/skills/fair-deal/  # (or .agents/skills/…) the skill itself, committed so Party B gets it on clone
  .gitignore                 # ignores private/ and the usual noise
  private/                   # GITIGNORED — never committed, never pushed. Local to each machine.
    whoami                   # one line: "A" or "B"
    solo-prep.md             # the Solo Prep Sheet: floor, BATNA, benchmarks, true goals/concerns
    notes/                   # private research, scratch reasoning, draft moves
```

`private/` exists in *both* clones at the same path, but holds different content (each owner's secrets) and is
never pushed — so neither side ever sees the other's floor. This is the firewall.

## state.json

```json
{
  "schemaVersion": 1,
  "parties": { "A": "<name or ''>", "B": "<name or ''>" },
  "phase": "setup | interview | negotiate | escalated | ready | accepted",
  "ready_to_negotiate": { "A": false, "B": false },
  "turn": "A | B",
  "round": 0,
  "round_cap": 12,
  "accepted": { "A": false, "B": false },
  "last_move": { "by": "", "at": "", "summary": "" }
}
```

- **phase** drives everything:
  - `interview` — each side is filling its private brief and seeding its opening canvas positions.
  - `negotiate` — begins only when `ready_to_negotiate.A` and `.B` are both true.
  - `escalated` — agents couldn't converge; humans must talk. (see `negotiate.md`)
  - `ready` — canvas is complete and internally agreed by the agents; awaiting human acceptance.
  - `accepted` — both `accepted.A` and `.B` true → `draft` may run.
- **turn** — whose move it is during `negotiate`. Only that side writes shared files.
- **round** — increments once per *exchange* (A then B = +1 is fine; simplest: increment whenever a side
  completes a move). Stop and escalate when `round >= round_cap`.

## The turn loop (every shared-file write follows this)

1. `git pull --rebase` (get the other side's latest move).
2. Read `state.json`. If `turn` ≠ me → it's not my turn: report status to my human, offer to keep polling
   (re-pull on an interval) or stop. Do **not** write shared files.
3. If `turn` == me → do my move (per the calling subcommand): update `canvas.json`, write
   `negotiation/from-<me>/round-<round>.md`, append a block to `negotiation/log.md`, update `state.json`
   (`turn` → other side, bump `round`, set `last_move`).
4. Stage only the files I'm allowed to write (my outbox, plus the shared `canvas.json`/`state.json`/`log.md`).
   **Never** stage `private/`.
5. `git add` → `git commit` → `git pull --rebase` → `git push`. If push is rejected, `git pull --rebase`
   and push again (someone's move landed first; re-read state and reconcile before retrying your move).

Because only the side whose `turn` it is writes shared files, steps 1–5 never truly collide; the rebase is a
safety net, not the mechanism.

## Commit message convention

`fair-deal: <phase>/<round> <by> — <short summary>`
e.g. `fair-deal: negotiate/3 B — counter on split; data fee off the top, 50/50 equity`

The commit log is the deal's provenance. Keep summaries factual and neutral (structure, not motive).

## What each side may write

| File | Writer |
|---|---|
| `canvas.json`, `negotiation/state.json`, `negotiation/log.md` | only the side whose `turn` it is |
| `negotiation/from-A/*` | only A |
| `negotiation/from-B/*` | only B |
| `private/*` | each side, locally, never committed |

## status (the `status` command)

Read `state.json` and `canvas.json` and report: phase, whose turn, round/round_cap, which boxes are filled vs
open, the canvas reading (Keep talking / Pilot first / Ready), and the single most useful next action. Never
read or reveal the other side's `private/` (you can't — it isn't in the repo).
